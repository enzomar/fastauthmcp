# Implementation Plan: IDP Resilience and Adapters

## Overview

This plan implements hardening features for the Ceramic framework: token exchange provider adapters (Google STS, Microsoft Entra ID), a circuit breaker for IDP HTTP calls, a resilient JWKS manager with request coalescing and stale-while-revalidate caching, and mutual TLS (mTLS) support for IDP communication. Tasks are ordered by dependency — foundational components first, then adapters, then integration, then tests.

## Tasks

- [x] 1. Verify existing tests pass with new optional fields.
- [x] 2. Implement CircuitBreaker state machine in `ceramic/resilience.py`: Create CircuitState enum (CLOSED, OPEN, HALF_OPEN). Implement CircuitBreaker class with asyncio.Lock for state transitions, `execute(coro_factory)` that checks state under lock and executes outside lock, `_record_success()` (reset failure count, HALF_OPEN→CLOSED), `_record_failure()` (increment count, open at threshold, HALF_OPEN→OPEN), `_is_failure(exc)` (True for RequestError and 5xx), `reset()` for testing, computed `state` property that checks cooldown expiry.
- [x] 3. Implement ResilientHttpClient in `ceramic/resilience.py`: Add class with constructor accepting CircuitBreaker and optional ssl_context. Implement `get(url, timeout, headers)` wrapping httpx in circuit breaker execute, `post_form(url, data, timeout)` for form-encoded POSTs, `post_token(url, body, timeout)` that calls post_form and parses response into TokenSet via _parse_token_response helper. Handle httpx exceptions by re-raising after circuit breaker records failure. Use ssl_context as httpx verify parameter when provided (for mTLS).
- [x] 3.1. Add `mtls` field (MtlsConfig|None) to AuthConfig. Add `get_mtls_ssl_context()` method to `TLSEnforcer` in `ceramic/security.py` that builds an ssl.SSLContext with client cert/key loaded and TLS 1.2+ enforced. Thread ssl_context through `ResilientHttpClient` (via `_verify` property), `OAuthService` (for raw httpx fallback calls in discover_endpoints and _post_token_request), `JWKSVerifier` (for JWKS discovery httpx call and PyJWKClient initialization), and `AuthenticationMiddleware` (passes to OAuthService and JWKSVerifier). Add `_build_mtls_context()` static method to `CeramicFastMCP` in `ceramic/server.py` that reads auth_config.mtls and returns an ssl.SSLContext or None. Wire mTLS context into AuthenticationMiddleware instantiation in `_build_pipeline()`. Validates cert/key file existence and raises ConfigurationError on missing/invalid files.

Implement `AdapterRegistry` in `ceramic/auth/adapters/registry.py` with register/get_adapter methods (None returns RFC8693, unknown raises ConfigurationError). Implement `RFC8693Adapter` in `ceramic/auth/adapters/rfc8693.py` (provider_id="rfc8693") that builds standard RFC 8693 body and POSTs via ResilientHttpClient.post_token, matching current OAuthService.token_exchange behavior exactly. ## Notes

- Tasks 2-3 (resilience layer) and 4-6 (adapters) can be developed in parallel after task 1.
- Task 3.1 (mTLS) depends on task 3 (ResilientHttpClient) since it adds ssl_context support to the client.
- Task 7 (JWKSManager) depends on task 3 (ResilientHttpClient) for its HTTP layer.
- Task 8 (OAuthService integration) is complete — includes both http_client and ssl_context parameters.
- Tasks 9 (middleware integration) depends on tasks 4-7 being complete.
- Task 10 (wiring) is the final integration point that brings all components together, including mTLS context creation.
- Tasks 11-13.1 (unit tests) can be written in parallel and should mock all external HTTP calls.
- Task 14 (integration tests + docs) is the final validation step.
- All new code should follow the existing project style: type hints, docstrings, async/await, httpx for HTTP.
- **File structure**: Each adapter lives in its own file under `ceramic/auth/adapters/` so they can evolve independently without merge conflicts or growing a single file too large. The package structure is:
  - `ceramic/auth/adapters/__init__.py` — re-exports public API (TokenExchangeAdapter, AdapterRegistry, all adapters)
  - `ceramic/auth/adapters/base.py` — TokenExchangeAdapter Protocol definition
  - `ceramic/auth/adapters/registry.py` — AdapterRegistry class
  - `ceramic/auth/adapters/rfc8693.py` — RFC8693Adapter (default)
  - `ceramic/auth/adapters/google.py` — GoogleSTSAdapter
  - `ceramic/auth/adapters/entra.py` — EntraOBOAdapter
